# AI TTS 服务接入

## 设计原则

对 Narratis 后端统一暴露 REST 调用，不强绑定某个具体推理框架。

## 许可证原则（必须阅读）

- 本项目代码是 `Apache-2.0`，不等于所有可接入模型都可商用。
- `Qwen3-TTS` 作为默认后端基线，优先保证开源和可持续分发。
- `Fish Audio` 仅作为可选 Provider 接入，按“需单独许可证审查”处理。
- 是否可商用、是否可再分发，以对应模型/服务官方最新条款为准。
- 详细规则见：[模型与服务许可证策略](./model-license-policy.md)。

## 当前后端类型

- `autodl`
- `local_vllm`
- `aliyun`

## 推荐接入流程

1. 先确定可用的 TTS Provider（本地/远程）。
2. 在设置页填写 Provider 所需地址和密钥。
3. 先跑角色试听链路，确认可用后再做批量合成。

## 零基础用户建议

- 优先提供“内置 sidecar + 自动检测 + 一键下载/启动”路线。
- 避免要求用户手配 Docker 和复杂 Python 环境。
- UI 中持续展示模型状态、下载进度、服务日志。

## 后续可扩展

- 新增 `provider_adapter` 抽象层（按 provider 解耦）。
- 增加 `/health` 检查和统一错误码，便于前端提示。
