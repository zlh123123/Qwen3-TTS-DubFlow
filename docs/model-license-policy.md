# 模型与服务许可证策略

本页用于明确 `Narratis` 项目许可证与第三方模型/服务许可证之间的边界。

## 核心原则

- 本仓库代码协议：`Apache License 2.0`
- 第三方模型权重、推理服务、云 API：遵循各自许可证与服务条款
- 任何第三方限制，不会因为本仓库是 Apache-2.0 而自动失效

## 默认与可选策略

### 默认后端（推荐）

- `Qwen3-TTS` 作为默认后端基线
- 目标：提供“开源可用、合规风险更低、可商用路径更清晰”的默认体验

### 可选后端（需单独确认）

- `Fish Audio` 等可选 Provider 仅作为插件式接入能力
- 在接入前必须自行确认其最新许可证条款（是否允许商用、再分发、托管、SaaS 等）
- Narratis UI 和文档会把这类后端标注为“需额外许可证审查”

## 分发与商用建议

- 对外发布桌面客户端时，默认只预置“许可证清晰”的后端选项
- 对“受限或不明确”许可证模型，不默认启用，不承诺商用可用
- 团队/公司场景建议保留一份内部《第三方模型合规清单》

本项目不提供法律意见。若你用于商业环境，请让法务或合规负责人审查你实际使用的模型和 API 条款。
